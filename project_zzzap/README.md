# Zzzap! — Developer Kickoff Pack 🚀

Привет, Куат! Я подготовил полный пакет документов для старта разработки бэкенда.

Эти файлы можно передать твоим разработчикам (CTO, тимлиду или фрилансерам), чтобы они не задавали лишних вопросов и сразу начали писать код.

## 📂 Что внутри?

### 1. `backend/schema.sql` (База Данных)
Готовая структура базы данных на PostgreSQL.
- **Пользователи:** Профили, аватарки, статусы (Gold).
- **Товары:** Видео, фото, цены, категории.
- **Заказы:** Покупки, статусы доставки, платежи.
- **Чат:** Сообщения между покупателем и продавцом.

### 2. `backend/api_spec.md` (API Маршруты)
Список всех запросов, которые фронтенд (твое мобильное приложение) будет отправлять на сервер.
- Вход/Регистрация.
- Лента (Feed).
- Создание товара.
- Покупка и оплата.

### 3. `../ZZZAP_ARCHITECTURE.md` (Общая Архитектура)
(Тот файл, что мы сделали ранее, лежит в корне workspace)
Описание всех экранов и пользовательских путей.

---

## 💡 Совет от "второго пилота":
Когда будешь нанимать бэкендера, скинь ему этот файл `schema.sql` и спроси:
*"Оцени, сколько времени займет поднять такой бэкенд на Node.js или Python (FastAPI) с учетом загрузки видео?"*

Если скажет **2-3 недели** на MVP — это адекватный срок.
Если **2 месяца** — многовато для старта.
Если **2 дня** — подозрительно быстро (значит, не сделает видео-процессинг нормально).

Удачи с кодом! ⚡️
